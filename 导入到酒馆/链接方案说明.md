# 脚本导入链接方案说明

## 问题说明

jsdelivr 提示 "Failed to fetch version info" 是因为：
- jsdelivr 需要 GitHub **Release**（发布版本）才能识别版本标签
- 仅创建标签（Tag）是不够的，需要创建正式的 Release

## 解决方案（按推荐顺序）

### 方案 1：使用提交哈希（当前推荐）✅

**优点**：最可靠，立即可用，无需额外操作

**链接**：
```javascript
import 'https://cdn.jsdelivr.net/gh/onlinedone/Cyber_prisoner_simulator@e5c84654a91e1c7793ff221e3b21d92a0a60b317/dist/赛博坐牢模拟器增强脚本/index.js'
```

**已更新到 JSON 文件中**

---

### 方案 2：使用 GitHub Raw 链接（备选方案）✅

**优点**：最直接，不依赖 CDN，立即可用

**链接**：
```javascript
import 'https://raw.githubusercontent.com/onlinedone/Cyber_prisoner_simulator/main/dist/赛博坐牢模拟器增强脚本/index.js'
```

**注意**：使用 `@main` 分支，文件更新后可能需要清除缓存

---

### 方案 3：使用 main 分支（临时方案）⚠️

**优点**：简单

**缺点**：可能有缓存问题，更新不及时

**链接**：
```javascript
import 'https://cdn.jsdelivr.net/gh/onlinedone/Cyber_prisoner_simulator@main/dist/赛博坐牢模拟器增强脚本/index.js'
```

---

### 方案 4：创建 GitHub Release（最佳长期方案）⭐

**步骤**：

1. 访问 GitHub 仓库：https://github.com/onlinedone/Cyber_prisoner_simulator

2. 点击右侧 **"Releases"** → **"Create a new release"**

3. 填写信息：
   - **Tag**: 选择 `v1.0.0`（或输入 `v1.0.0` 创建新标签）
   - **Release title**: `版本 1.0.0：完整系统`
   - **Description**: 
     ```
     完整系统：核心系统、状态栏、事件系统、NPC系统、知识库加载器
     
     功能说明：
     - 核心系统：提供基础功能和模块管理
     - 状态栏系统：状态追踪、HTML注释解析、缓慢变化机制
     - 事件系统：按日推进、法律流程、随机事件
     - NPC系统：NPC生成和管理
     - 知识库加载器：动态加载知识库
     ```

4. 点击 **"Publish release"**

5. 等待 5-10 分钟让 jsdelivr 同步

6. 然后可以使用版本标签链接：
   ```javascript
   import 'https://cdn.jsdelivr.net/gh/onlinedone/Cyber_prisoner_simulator@v1.0.0/dist/赛博坐牢模拟器增强脚本/index.js'
   ```

---

如果方案 1 仍然无法访问，请使用 **方案 2（GitHub Raw）**。

## 更新脚本时的注意事项

- **使用提交哈希**：每次更新后需要获取新的提交哈希并更新链接
- **使用 GitHub Raw**：使用 `@main` 分支，更新后可能需要清除浏览器缓存
- **使用 Release**：创建新 Release 后，更新链接中的版本号即可
