# jsdelivr 链接问题解决方案

## 问题原因

jsdelivr 在处理包含中文字符的文件路径时可能出现问题，导致无法正确访问文件。

## 解决方案（按推荐顺序）

### ✅ 方案 1：使用 GitHub Raw 链接（当前推荐）

**优点**：
- ✅ 最可靠，不依赖 CDN
- ✅ 支持中文路径
- ✅ 立即可用，无需等待同步

**链接**：
```javascript
import 'https://raw.githubusercontent.com/onlinedone/Cyber_prisoner_simulator/main/dist/赛博坐牢模拟器增强脚本/index.js'
```

**已更新到 JSON 文件中**

---

### 方案 2：使用提交哈希（备选）

如果 GitHub Raw 也无法访问，可以使用提交哈希：

```javascript
import 'https://cdn.jsdelivr.net/gh/onlinedone/Cyber_prisoner_simulator@e5c84654a91e1c7793ff221e3b21d92a0a60b317/dist/赛博坐牢模拟器增强脚本/index.js'
```

---

### 方案 3：使用不同的 jsdelivr 镜像

如果在中国大陆，可以尝试不同的 CDN 镜像：

**Fastly 镜像**：
```javascript
import 'https://fastly.jsdelivr.net/gh/onlinedone/Cyber_prisoner_simulator@v1.0.0/dist/赛博坐牢模拟器增强脚本/index.js'
```

**Gcore 镜像**：
```javascript
import 'https://gcore.jsdelivr.net/gh/onlinedone/Cyber_prisoner_simulator@v1.0.0/dist/赛博坐牢模拟器增强脚本/index.js'
```

---

### 方案 4：URL 编码路径（如果必须使用 jsdelivr）

如果必须使用 jsdelivr，可以尝试对中文路径进行 URL 编码：

```javascript
// 原始路径：dist/赛博坐牢模拟器增强脚本/index.js
// URL 编码后：
import 'https://cdn.jsdelivr.net/gh/onlinedone/Cyber_prisoner_simulator@v1.0.0/dist/%E8%B5%9B%E5%8D%9A%E5%9D%90%E7%89%A2%E6%A8%A1%E6%8B%9F%E5%99%A8%E5%A2%9E%E5%BC%BA%E8%84%9A%E6%9C%AC/index.js'
```

## 为什么 GitHub Raw 更可靠？

1. **不依赖 CDN**：直接访问 GitHub 服务器，无需等待 CDN 同步
2. **支持中文路径**：GitHub 原生支持中文路径，无需编码
3. **即时更新**：代码更新后立即生效，无需等待缓存
4. **全球可用**：GitHub 在全球都有节点，访问速度快

---

## 注意事项

- GitHub Raw 链接使用 `@main` 分支，代码更新后会自动使用最新版本
- 如果需要固定版本，可以使用提交哈希或创建 Release 后使用版本标签
- 如果担心 GitHub 访问问题，可以考虑使用自建服务器或对象存储服务
