# GitHub 仓库可见性说明

## Release 可见性规则

**重要**：GitHub Release 的可见性与仓库的可见性一致：

- ✅ **公开仓库（Public）**：Release 对所有人可见
- 🔒 **私有仓库（Private）**：Release 只对协作者可见

## 检查仓库可见性

访问你的仓库：https://github.com/onlinedone/Cyber_prisoner_simulator

查看仓库页面顶部：
- 显示 **"Public"** → 公开仓库，所有人可见
- 显示 **"Private"** → 私有仓库，仅协作者可见

## 不同可见性的影响

### 公开仓库（Public）

**优点**：
- ✅ jsdelivr CDN 可以正常访问
- ✅ 任何人都可以使用脚本链接
- ✅ 便于分享和协作

**缺点**：
- ⚠️ 代码对所有人可见
- ⚠️ 任何人都可以查看代码和提交历史

**适用场景**：
- 开源项目
- 不涉及敏感信息的代码
- 需要公开分享的脚本

### 私有仓库（Private）

**优点**：
- ✅ 代码不公开，保护隐私
- ✅ 只有协作者可以访问

**缺点**：
- ❌ jsdelivr **无法访问**私有仓库的文件
- ❌ 需要使用其他方式托管文件（自建服务器、私有 CDN 等）
- ❌ 无法直接通过公开链接分享

**适用场景**：
- 包含敏感信息的代码
- 商业项目
- 需要保密的脚本

## 如何更改仓库可见性

### 改为私有

1. 访问仓库：https://github.com/onlinedone/Cyber_prisoner_simulator
2. 点击 **"Settings"**（设置）
3. 滚动到最下方 **"Danger Zone"**（危险区域）
4. 点击 **"Change visibility"**（更改可见性）
5. 选择 **"Make private"**（设为私有）
6. 确认操作

### 改为公开

1. 访问仓库设置页面
2. 在 **"Danger Zone"** 中点击 **"Change visibility"**
3. 选择 **"Make public"**（设为公开）
4. 确认操作

## 私有仓库的替代方案

如果必须使用私有仓库，但需要公开访问脚本文件，可以考虑：

### 方案 1：自建服务器

1. 将编译后的文件上传到自己的服务器
2. 使用自己的服务器链接：
   ```javascript
   import 'https://your-server.com/scripts/detention-system/index.js'
   ```

### 方案 2：使用对象存储服务

1. 使用阿里云 OSS、腾讯云 COS、AWS S3 等
2. 设置公开读取权限
3. 使用对象存储的公开链接

### 方案 3：使用 GitHub Pages（仅限公开仓库）

1. 在仓库设置中启用 GitHub Pages
2. 将编译后的文件放到 `gh-pages` 分支
3. 使用 GitHub Pages 链接访问

## 推荐方案

**对于看守所模拟器脚本**：

- ✅ **推荐保持公开**：代码不涉及敏感信息，公开便于使用 jsdelivr
- ✅ 如果必须私有：使用自建服务器或对象存储服务托管编译后的文件

## 注意事项

1. **jsdelivr 不支持私有仓库**：即使创建了 Release，私有仓库的文件也无法通过 jsdelivr 访问
2. **GitHub Raw 链接也需要认证**：私有仓库的 raw 链接需要登录 GitHub 才能访问
3. **公开仓库的代码可以被 fork**：其他人可以复制你的仓库
4. **提交历史可见**：公开仓库的所有提交历史都对所有人可见
